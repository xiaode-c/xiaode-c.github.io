python之禅

发表于 2016-05-08   |   分类于 python   |     |  

想必了解python的人，都知道在python的交互式命令行中，执行import this,会出现一首诗，这就是python之禅，我们应该争取能使自己做到以下这些

注意: 以下是我的翻译，由于水平有限，可能有很多错误，慎重！！！

美丽胜于丑陋
显式优于隐式
简洁胜于复杂
复杂胜于繁琐
扁平胜于嵌套
间隔胜于紧凑
可读性很重要
即便在特例的实用的情况下，也不应该违背这些规则
不要包容所有错误，除非你确定需要这样做
当存在多种可能，不要尝试去猜测
应该有一个 — — 并且最好只有一个 — — 明显的解决办法。
虽然这并不容易，因为你不是 Python 之父
如果你无法向别人描述你的方案，他就是个坏主意
如果很容易表述，那就是一个好方案
命名空间是一种绝妙的理念，我们应当多加利用
